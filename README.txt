-简介
    这是一个任务编辑器的脚本编辑框架，其功能是允许编辑者通过简单配置来使游玩任务的玩家可以通过额外无线电菜单来使用一个数据链，这个数据链的特性有：
        1，可以指挥ai组群执行侦查、锁定目标、激光照射、定点打击、猎歼任务
        2，任务中可以同时存在多个数据链实例且互不干扰，通过“rangeReference"这个table来定义本数据链实例的指挥控制者是谁(阵营中所有玩家均可控制还是仅指定群组内的玩家可以控制)
-配置方法
    -如何部署框架
        根目录下的两个lua文件分别是ERD框架，mist
        你需要编写的是一个数据链定义脚本
        本框架依赖于mist，部署框架的方法是在任务编辑器中设置一个置于第一位的“仅一次-当任务开始”的无条件触发器，触发器动作为三条“执行脚本文件”动作 依次为 执行mist 执行ERD框架 执行你的数据链定义脚本
        ---------不要弄错执行顺序---------
        ---------不要执行sample文件夹里的lua---------
    -如何编辑你的数据链定义脚本
        参考sample文件夹中的sample.lua
-在游戏中如何使用这个数据链
    -如果你正确进行了配置和定义，那么具有控制权的组群应该可以通过斜杠无线电菜单中的f10（“其它”）来访问数据链面板
    -数据链的基本运作方式是“定义SPI坐标点”然后“执行操作”，一个数据链实例在任一游戏时刻只存在至多一个SPI，各种可执行的操作（若需要围绕某个目标来执行）都将针对这个SPI坐标点执行
    -定义SPI
        -通过f10地图用户标记定义SPI
            -打开f10地图，左上角选项卡中有“用户标记”这一选项，点击它后在地图上左键点按即可放下用户标记
            -用户标记是在阵营这一层面共享的，因此同一阵营中不同组群放下的用户标记可能会造成互相干扰，这一点受制于游戏特性，暂时没有好的解决办法
            -数据链主菜单中的“选择地图标记”选项将会把最后一个同阵营成员放下的用户标记在地面上的投影点定义为SPI
            -你还可以打开自动获取地图坐标为SPI的开关选项，这会使任何同阵营成员放下一个用户标记时，你的SPI都会自动更新为该标记坐标
                -如果你通过其它方式获取了SPI，这个开关会自动关闭，你也可以通过同一个开关选项来手动关闭它
            -获取的用户标记不受删除用户标记影响，即这些方法总是试图获取最后一个同阵营成员放下的标记的坐标，即使这个标记已经被删除了
        -通过侦查跟踪来定义SPI
            -如果你的数据链中存在一个具有侦查能力的受控组群，你指挥它进行侦查后它侦查到的目标将被列在一个子菜单中，你可以通过选择该子菜单中的条目来要求该受控组群尝试连续跟踪它
                -SPI将持续保持在该目标身上，直到你选择其它SPI数据源或者跟踪丢失
                -丢失跟踪时，SPI将停留在最后已知位置
